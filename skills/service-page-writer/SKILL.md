# Service Page Writer

Write a page that helps a client decide.

## Workflow

1. Define the service and buyer.
2. Clarify the buyer's problem and outcome.
3. Structure the page in a buying-friendly order.
4. Explain the service clearly.
5. Add process, proof, FAQ, and CTA.
6. End with the page's weakest missing proof or clarity gap.

## Rules

- Keep the service promise specific.
- Write for the buyer, not for the seller.
- Explain what happens and why it matters.
- Make the CTA explicit.

## Default structure

- headline
- problem
- solution
- benefits
- process
- proof
- FAQ
- CTA
